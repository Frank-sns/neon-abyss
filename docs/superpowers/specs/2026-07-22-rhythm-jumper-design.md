# 节奏跳跃跑酷游戏 (Rhythm Jumper) — 设计文档

## 概述

Chrome 小恐龙风格的网页跑酷游戏。玩家控制一个赛博朋克风格的角色在霓虹夜跑场景中跳跃躲避障碍物，挑战最高分。单文件 HTML，浏览器打开即玩。

## 技术架构

- **单文件 HTML** — HTML + CSS + JavaScript 全部内联，零外部依赖
- **Canvas 渲染** — 使用 `<canvas>` + 2D API 绘制所有游戏元素
- **本地启动** — `python -m http.server` 或直接用浏览器打开文件

## 游戏机制

### 角色操作
- 空格键/上箭头：跳跃
- 空中可再按一次触发二段跳（带短暂冷却提示）
- 角色固定在屏幕左侧 20% 处水平位置

### 障碍物
- 从屏幕右侧匀速逼近，高度在 2-3 个预设值中随机
- 外观为暗红色尖刺方块，带警示光晕
- 障碍物间距随速度增加略微减小

### 难度递增
- 每存活 10 秒，障碍物移动速度提升一档
- 速度上限为初始速度的 3 倍
- 速度变化通过地面网格线速度同步体现

### 计分规则
- 存活时间 × 10 = 基础分数（每秒 10 分）
- 每安全越过一个障碍物：+50 分
- 游戏内实时显示当前得分

### 结束与重来
- 角色碰撞到障碍物 → 游戏结束
- 弹出结算面板：本次得分 + 历史最高分
- 按空格键或点击按钮重新开始
- 最高分存储在 `localStorage`

## 视觉风格

### 配色方案（赛博朋克夜跑）
- 背景：深蓝黑 `#0a0a1a` → 深紫 `#1a0a2e` 渐变
- 地面/网格线：霓虹青 `#00f0ff`
- 角色：亮白 + 青色光晕，跳跃时拖尾粒子
- 障碍物：暗红 `#cc2233` + 红色光晕 `rgba(255, 0, 50, 0.6)`
- UI 文字：白色，数字用霓虹黄色 `#ffd700`

### 元素设计
- **角色**：20×30 像素的圆角方块，带有发光边框和底部光晕
- **粒子拖尾**：跳跃时从角色底部洒出 5-8 个青色小粒子，向上飘散消失
- **障碍物**：25×35 像素的三角/尖刺形状，顶部尖锐，有红色脉冲光晕
- **地面**：一条水平线 + 向右移动的虚线网格标记，速度与障碍物同步
- **背景**：随机散布的远处霓虹光点（"城市灯光"），视差慢速移动
- **分数 UI**：右上角，大号霓虹黄色字体，轻微发光效果
- **结算面板**：居中半透明暗色面板，圆角，显示分数和重新开始提示

### 音效（可选，Web Audio API）
- 跳跃时：短促上升音阶（~0.1s，800→1200Hz）
- 二段跳：更高音阶（~0.1s，1200→1600Hz）
- 碰撞：低频噪声脉冲（~0.3s，200Hz 方波衰减）
- 过障碍：叮咚提示音（~0.15s，600Hz 正弦波）

## 游戏循环

```
1. 更新背景视差元素位置
2. 更新障碍物位置、检测出屏移除
3. 更新角色物理（重力、跳跃状态、粒子）
4. 碰撞检测（角色 AABB vs 障碍物 AABB）
5. 生成新障碍物（间距随机）
6. 更新分数和难度
7. 渲染全部元素
8. requestAnimationFrame 循环
```

## 边界条件

- 窗口大小自适应：Canvas 按 `window.innerWidth/innerHeight` 设置，监听 resize 事件
- 最低可玩分辨率：320×480
- 移动端：显示"请用电脑打开获得最佳体验"提示（初版不做移动适配）

## 测试要点

- 跳跃高度能越过最高障碍物
- 二段跳能越过连续两个障碍物
- 难度增长速度合理，不会瞬间无法玩
- 碰撞检测准确，无明显"穿模"
- localStorage 最高分正常读写
- 游戏结束后重开状态完全重置
